# 改修仕様書 03

## フェーズ1.3 - バックエンドAPIルート強化

### 改修の目的
Zaico APIへのアクセスをバックエンド経由に変更し、APIキーの露出を防ぐ

### 改修内容

#### 1. 既存APIルートの強化
- `api/zaico/[...path].js`の拡張
- 環境変数からのAPIキー取得
- エラーハンドリングの改善

#### 2. APIキー管理機能
```javascript
// api/zaico/[...path].js
const ZAICO_API_KEY = process.env.ZAICO_API_KEY;

export default async function handler(req, res) {
  // APIキー検証
  // リクエストプロキシ
  // エラーハンドリング
}
```

#### 3. レスポンスフォーマットの統一
- 成功レスポンスの標準化
- エラーレスポンスの標準化

### 影響範囲
- `api/zaico/[...path].js`

### 所要時間
3-4時間

### 検証項目
- Zaico APIへのプロキシが正常動作すること
- エラーレスポンスが適切に返されること
- APIキーが環境変数から正しく読み込まれること
