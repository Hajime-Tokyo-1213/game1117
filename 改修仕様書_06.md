# 改修仕様書 06

## フェーズ1.6 - ApiKeyCheckerコンポーネント修正

### 改修の目的
APIキーチェック機能をバックエンド経由に変更する

### 改修内容

#### 1. ApiKeyChecker.jsxの修正
- localStorageからのAPIキー取得を削除
- バックエンド経由でのAPIキー状態確認

#### 2. APIキー状態確認API
```javascript
// api/zaico/status.js
export default async function handler(req, res) {
  const apiKey = process.env.ZAICO_API_KEY;
  
  if (!apiKey) {
    return res.status(500).json({ configured: false });
  }
  
  // APIキーの有効性テスト
  return res.json({ configured: true, valid: true });
}
```

#### 3. UIの更新
- APIキー入力フィールドの削除または非表示
- 設定状態の表示のみ

### 影響範囲
- `src/components/ApiKeyChecker.jsx`
- 新規: `api/zaico/status.js`

### 所要時間
2-3時間

### 検証項目
- APIキー状態が正確に表示されること
- APIキーの入力ができなくなっていること
- 管理者のみが設定状態を確認できること
